HW 2 [10%].

Instructions:

  • Team Allowed. Maximum 2 per Team.
  • State Clearly team members name and IDs as comments in source codes and each page of submitted report.
  • Points 100 (Grade 10%).
  • There are 2 parts to this assignment.
  • Due Date: 10.11.2012, 11:59:59 am EST (Before Class Begins)
  • How to submit: A pdf of the report along with modified ".asm" file should be sent to anaik3@gmu.edu and rangwala@cs.gmu.edu with the subject "[CS 465] HW2".
  • Plagiarism is not permitted in any form. I enforce the university policy and honor code.
  • Have Fun. Discuss on Piazza but don't give away answers.

MIPS Programming (Part 1 - 50 points)


Russian Peasant Multiplication : The goal of the program is to write a function that implements Russian Multiplication. Your code should be as efficient as possible and should use only MIPS basic instructions. No "mul" is permitted.

See the video for an example of how the Russian Peasant Multiplication works.

Program Parameters:

  1. The program should accept two input signed integers and call a procedure called Russian_Multiplication.
  2. Result should be returned from the procedure Russian_Multiplication and should be displayed on the output screen/console.
  3. You may have to write a procedure to figure out the correct sign after multiplication.
  4. We will assume that the absolute value of the numbers will lie between -2^14 .. 2^14
  5. Feel free to use the skeleton file from HW1.

Examples and Test Cases:

  1. Input Number 1: 53
  2. Input Number 2: -89
  3. Output: -4717.

Deliverables:

  1. Source Code: russian_XXX.asm where XXX is your "last names combined".

Exercises (Part 2 - 50 points)

Instructions: Show workings to get partial credit. Answers need to be SCANNED or typedin a word processor like MS Office, Notepad or latex. Final submission should be a PDF called "hw2_xxx.pdf", where xxx = last names combined.
Question 1. (15 points)

Consider the following hypothetical MIPS instructions.

  • A. abs $t2, $t3 # R[rd] = | R[rs] |
  • B. sgt $t1, $t2, $t3 # R[rd] = (R[rs] > R[rt]) ? 1.0

Answer for each of the two instructions.

  • 1.i) Why are these instructions not included in the MIPS instructions set ?
  • 1.ii) Write the shortest sequence of MIPS instruction instruction to perform these operations.
  • 1.iii) If these instructions, were implemented in MIPS which instruction format would you use and why ?
Question 2. (10 points)

Consider the following MIPS Code:

LOOP: slt $t2, 0, $t1
beq $t2, 0, DONE
subi $t1, $t1, 1
addi $s2, $s2, 2
j LOOP
DONE:

Answer the following questions:

  • 2.i) Assuming that $t1 = 10 and $s2 = 0 initially. What is the final value of $s2?
  • 2.ii) Write the C code for the MIPS code assuming that registers $s2, $t1 and $t2 are x, y and z, respectively.
  • 2.iii)For the MIPS Assembly code, assuming that $t1 is initialized to N . How many MIPS instructions will be executed ?
Question 3 (25 points).
  • 3.i) If the PC is at address 0x 0000 0000 . How many branch (no jumps allowed) do you need to get to the address 0x0002 0000
  • 3.ii) If the PC is at address 0x 0000 0000 . How many jump instructions (no jump registers or branches allowed) do you need to get to the address 0x0002 0000
  • 3.iii) If the PC is at address 0x 00000000 . How many jump (no jump registers allowed) do you need to get to the address FFFF FF00
  • 3.iv) MIPS Architecture designers have decided to modify the immediate field of I-type instructions from 16 bits to 8 bits. In that scenario, if the PC is at address 0x 0000 0000 , how many branch instructions are needed to get to the address 0x0002 0000 .
  • 3.v) MIPS Architecture designers have decided to modify the immediate field of I-type instructions from 16 bits to 8 bits. In that scenario, if the PC is at address 0x 0000 0000 , how many jump instructions are needed to get to the address 0x0002 0000 .
  • 3.vi) What are the five different types of addressing that are implemented in MIPS ? Give one example of each.

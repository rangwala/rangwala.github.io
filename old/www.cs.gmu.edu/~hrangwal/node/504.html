MIPS Programming (Part 1 - 50 points)


Towers of Hanoi (Picture Courtesy: Wikipedia)

The tower of Hanoi is a puzzle that involves three towers (labeled A, B, C) (or place holders) which has n discs of varying sizes that need to be moved from the orignal tower to the third tower.

The rules for the puzzle are the following:

  1. You may move only one disc at a time.
  2. At no time may a smaller disc be underneath any of the larger discs.

Write a MIPS code that will print instructions on how to move from one tower to another, step-by-step, using an arbitrary number of disks. The program should accept as input any value "n" for the number of disks from the prompt. Refer to previous HWs to see how to accept inputs in MIPS. All disks are assumed to start at Tower A.

It is essential that you think and write a recursive code for this implementation.

Below is an example of how the output should look like (assuming n =5)


Number of disks = 5.

Move disk from A to B
Move disk from A to C
Move disk from B to C
..
..
..

Deliverables:

  1. Source Code: CS465_HW3_XXX.asm where XXX is your "last name".
  2. Print your output for n=10. Count the number of instructions executed by the MIPS simulator and state this in the PDF of the report.
